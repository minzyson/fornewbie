대학정보공시란?

대학정보공시란?

대학이 교육현황, 취업률, 재정 현황 등 대학 운영의 핵심 정보를 국민 누구나 열람할 수 있도록 대학알리미(academyinfo.go.kr)에 게시하는 것을 의미합니다.

업무 개요

대학정보공시는 14개 분야, 65개 항목, 103개 세부항목으로 구성되어 있습니다. 처음 대학정보공시 업무를 맡으면 항목이 많아 부담스럽게 느껴질 수 있습니다.  하지만 대학정보공시는 한 사람이 모든 항목을 직접 입력하고 처리하는 업무가 아닙니다. 입력담당자는 본인의 담당 공시 항목을 숙지하여 기한 내 공시 데이터를 검증하면 됩니다.

고등교육통계란?

고등교육통계조사란?

한국교육개발원(KEDI)이 주관하는 국가 승인 통계로, 전국 고등교육기관의 학생·교원·직원·강좌 현황을 조사·제출하는 업무입니다.
수집된 통계 데이터는 교육부의 고등교육 정책 수립, 국내외 교육 비교 분석, 그리고 대학정보공시(대학알리미)의 공시 항목으로 직접 연계됩니다.

이 업무, 왜 정확해야 할까요?

고등교육통계조사에 입력한 데이터는 단순 통계로 끝나지 않습니다.
고등교육통계 → 대학정보공시(대학알리미)로 자동 연계되는 항목이 매우 많습니다.
즉, 우리 대학이 제출한 원자료의 오류가 그대로 공개 공시에 반영될 수 있습니다.

고등교육통계조사 일정
상반기 조사
2026년 4월 1일 기준
· 원자료 서식 확인: 3월

· 입력·공시연계장표 제출: 4.13~4.16

하반기 조사
2026년 10월 1일 기준
· 10월 기준일 원자료 등록
· 데이터 검증 및 제출
· 일정은 상반기와 동일한 흐름
· 공문 수령 후 총괄담당자 안내에 따라 진행

고등교육통계는 연 2회 시행되며, 매년 4월 1일과 10월 1일을 기준시점으로 조사가 진행됩니다. 기준일에 발생하는 학적·임용 등 변동사항을 반영하기 어려우므로, 기준일 이전에 관련 데이터 정비가 반드시 완료되어야 합니다.

꼭 알아야 할 핵심 용어
원자료
UWIN 시스템에서 다운받는 엑셀 자료. 우리 대학 UWIN 데이터를 통계조사 포맷에 맞춰 출력한 것으로, 고등교육통계 시스템에 업로드하는 기초 파일입니다.
경로: UWIN → 일반행정 → 기획관리 → 통계조사관리 → 고등교육통계조사(2023~)
기초집계
우리 대학이 제출한 원자료를 바탕으로 한국교육개발원이 시스템을 통해 자동으로 집계한 장표입니다. 담당 부서는 집계된 수치가 정확한지 확인해야 합니다.
입력장표
원자료에서 자동 집계되지 않아 한국교육개발원이 각 대학에 직접 집계치 입력을 요구하는 장표입니다. 담당 부서가 직접 수치를 입력·검증해야 합니다.
공시연계장표
고등교육통계 → 대학정보공시(대학알리미)로 자동 연계되는 장표입니다. 이 장표의 오류는 공개 공시에 그대로 반영되므로 가장 주의가 필요한 항목입니다.
고등교육통계조사 업무 흐름 (7단계)
1
원자료 생성 및 등록
UWIN에서 원자료(엑셀) 다운 → 고등교육통계조사 시스템(hi.kedi.re.kr)에 업로드
2
교차검증
원자료 항목 간 데이터 불일치 여부 확인. 교차검증이 완료되어야 다음 단계(기초집계) 진행 가능합니다. (예: 입학정보 ↔ 교류정보)
3
기초집계
원자료 기반으로 KEDI가 자동 집계한 장표 수치 확인. 전 부서의 기초집계가 완료되어야 입력장표 입력이 가능합니다.
4
내용검증
집계 수치의 논리적 오류 검출 단계. 오류 발생 시 사유 등록 또는 원자료 수정으로 처리합니다.
5
입력장표 제출
입력장표는 자동 집계가 불가능한 항목으로, 담당 부서에서 관련 데이터를 시스템에 직접 입력하여야 합니다. 자료 입력은 화면에서 직접 입력하거나 엑셀 서식 파일을 작성하여 업로드하는 방식으로 진행할 수 있으며, 입력 완료 후 내용검증을 거쳐 제출하여 주시기 바랍니다.
6
공시연계장표 제출 가장 주의!
해당 장표는 고등교육통계 입력자료가 대학정보공시로 직접 연계되는 항목입니다. 입력 오류가 있을 경우 공개 공시자료에 그대로 반영될 수 있으므로, 제출 전 관련 수치를 반드시 검증하여 주시기 바랍니다.
7
최종제출 (기획평가팀)
전 부서의 자료 제출이 완료된 후, 기획평가팀에서 전체 입력자료에 대한 최종 기초검증을 수행하고 KEDI에 최종 제출합니다.

처음 해야 할 일

대학정보공시시스템 회원가입

대학정보공시 회원가입을 위해서는 공인인증서가 필요합니다.  (개인금융인증서, EPKI, GPKI도 가능) 공인인증서가 없는 경우에는 교육부 공인인증서를 별도 발급받아야 합니다. 기존 공인인증서(금융인증서 포함)가 있으면 STEP 2로 바로 이동하세요!
STEP 1
교육부 인증서 발급
1

UWIN 로그인 홈페이지 → 안내사항 바로가기 클릭

UWIN 로그인 화면 우측 하단 안내사항 목록에서 [안내사항 바로가기] 버튼 클릭

2

교육부 인증서 발급 신청서 다운로드 후 기재

안내사항 화면에서 [인증서 발급 신청서 다운로드] 클릭 → 양식 작성

3

정보화개발팀 담당자에게 이메일 송부

작성한 신청서를 이메일로도 제출합니다.

담당자: 방지현  |  bjh1020@ulsan.ac.kr
4

승인 완료 후 로그인 가능 여부 확인

승인 처리 완료 후 인증서로 로그인 가능한지 확인합니다.

STEP 2
대학정보공시 시스템 회원가입
1

대학알리미 접속 → 업무담당자 로그인

academyinfo.go.kr 접속 → 우측 하단 [업무담당자 로그인] 버튼 클릭

2

사용자 등록 클릭 → 이용약관 동의

[사용자 등록] 클릭 → 이용약관 및 개인정보 수집 동의 체크 → [사용자 등록] 버튼 클릭

3

인증서 선택 후 확인

KSIGN 창에서 사용할 인증서 선택 → 인증서 암호 입력 → [확인]

4

사용자 정보 입력 → 확인

ID 입력 ① → 업무 구분 선택 ② → 소속대학 검색 ③ → 이메일 인증 ④ → 인증번호 입력 ⑤ → [확인] ⑥ 클릭

업무 구분은 대학정보공시 및 조준별(학교도입력) 선택
5

기획평가팀 총괄 담당자(손민지, 1166)에게 반드시 공유

회원가입 완료 후, 시스템 권한 부여를 위해 기획평가팀 총괄 담당자에게 반드시 알려주세요.

처음 해야 할 일

고등교육통계시스템 회원가입

예전에 가입한 적 있다면?
회원가입을 다시 하지 않으셔도 됩니다.
아래 STEP 2의 권한 부여 절차만 확인해 주세요.
고등교육통계시스템이 처음이라면?
아래 절차대로 사용자 등록을 진행해 주세요.
STEP 1
개인정보보호 교육 이수 필수
고등교육통계 업무 담당자(학생·교원 기본정보 입력)는 회원 가입을 위해 개인정보보호 교육 이수증을 반드시 등록해야 합니다.
미등록 시 총괄담당자는 입력담당자 승인·권한 부여 화면이 뜨지 않고, 입력담당자는 학생·교원 기본정보 화면에 접근할 수 없습니다.
1

개인정보배움터 접속 및 회원가입

아래 경로로 접속해 회원가입 후 교육과정을 수강합니다.

교육과정 → 온라인교육(개인) → 개인정보처리자 → 개인정보배움터
개인정보배움터 바로가기 ↗
2

수료증 저장

교육 완료 후 수료증을 PDF 또는 JPG/PNG 형식으로 저장해 두세요. 다음 단계에서 업로드합니다.

파일명 권장 형식: 성명_소속대학_수료증 / 5MB 이하
STEP 2
시스템 접속 → 사용자 등록
1

hisys.kedi.re.kr 접속 → [사용자 등록] 클릭

신규 담당자 또는 전년도 하반기 미가입자는 [사용자 등록] 버튼 클릭

2

이용약관 동의 → 인증서 선택 → [사용자 등록] 클릭

① 이용약관 확인 및 동의 체크 → ② [사용자 등록] 클릭 → ③ 인증서 팝업에서 인증서 선택 후 암호 입력 → [확인]

3

정보입력 → 이수증 업로드 → [신청] 클릭

필수항목 입력: ID · 성명 · 기관분류(고등교육기관) · 사용자구분(학교 총괄/입력담당자) · 학교선택 · 소속부서 · 이메일 · 휴대전화

개인정보 취급담당자는 개인정보 취급 여부 → 예 선택 후 STEP 1에서 받은 이수증 파일 업로드 필수

기관분류·사용자구분·학교선택 오입력 시 가입이 반려될 수 있습니다.
4

총괄담당자(손민지, 1166)에게 반드시 공유

가입 신청 후 총괄담당자가 승인 + 권한 부여를 해야 시스템 로그인이 가능합니다. 신청 완료 후 반드시 알려주세요.

인증서를 바꿔야 하는 경우

인증서 재발급(갱신) 또는 다른 인증서로 변경(교육부 ↔ 금융) 시 메인 화면에서 [인증서 변경] 클릭

① 기존 등록 정보(ID·성명·휴대전화·이메일) 입력 → ② [회원가입 여부 확인] → PASS 또는 문자 인증 → ③ [인증서 변경] 클릭 후 새 인증서 선택
